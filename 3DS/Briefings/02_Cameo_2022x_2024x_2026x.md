# Cameo 2022x / 2024x / 2026x

## What Cameo is

Cameo Systems Modeler is the systems engineering/MBSE client built on the MagicDraw platform. In current 3DS naming, the related product branding includes:
- Magic Cyber Systems Engineer / Cameo Systems Modeler
- MagicDraw platform underneath
- Plugins and companion products layered on top

The practical rule:
- If you are automating the modeling client, you are usually standing on MagicDraw APIs and Cameo/SysML plugins.

## Release line view

### 2022x
What stands out:
- SysML 1.7 support was already called out in 2022x.
- Server-side simulation showed up as a headline feature.
- Reader mode was introduced.
- Classified/sensitive-data workflow support was expanded via the Data Markings and Classification Plugin.

### 2022x Refresh2
What stands out:
- Better MagicGrid package-structure automation.
- More server-side simulation capabilities.
- FMU generation/export improvements.
- Better Excel sync options.
- Performance improvements on both client and Teamwork Cloud sides.

### 2024x
What stands out:
- Stronger collaborative-modeling and branch-activity support around the ecosystem.
- Continued simulation/reporting/documentation improvements.

### 2024x Refresh3
What stands out:
- Results Player improvements.
- Better FMU support, including token use during FMU generation.
- CSV export/state-visualization enhancements.
- Collaborative-modeling improvements, including partial project usages and improved SSO support across command-line tools.

### 2026x Refresh1
What stands out:
- More SysML v1 modeling and simulation improvements.
- Better Excel import, including nested connectors.
- Partial Project Usages matured beyond technology preview.
- Improved earlier-version save/export behavior for projects, including TWC projects to `.mdzip`.

## AI mental model for Cameo

Think in layers:
1. Platform layer
   - MagicDraw core
2. Language/plugin layer
   - SysML v1 plugin
   - SysML v2 plugin where supported
   - CST, DataHub, Report Wizard, etc.
3. Repository/collab layer
   - local `.mdzip`
   - Teamwork Cloud
   - 3DEXPERIENCE integrations in some lines
4. Automation layer
   - Open Java API plugins
   - Report Wizard templates
   - Groovy/report scripts
   - REST/OSLC through Teamwork Cloud

## What changed across generations

From 2022x to 2026x, the trend is:
- stronger collaborative modeling
- deeper repository integrations
- more mature simulation/export/reporting
- gradual movement toward SysML v2 support in the wider portfolio
- more branch/resource/project usage awareness

## Operational heuristics

### If the goal is modeling work
- Use the correct SysML plugin/language line first.
- Confirm the edition and release family before assuming feature availability.

### If the goal is extraction/reporting
- Prefer:
  - Report Wizard
  - Open API plugin
  - Teamwork Cloud REST/OSLC
  - export functions
- Avoid direct reverse-parsing of zipped project files unless the workflow is read-only and controlled.

### If the goal is migration
- Separate these concerns:
  - model content migration
  - profile/plugin compatibility
  - repository upgrade path
  - Java/runtime compatibility

## AI "do not confuse these" list

- Cameo Systems Modeler is not the same thing as Teamwork Cloud.
- SysML v1 support is not SysML v2 support.
- MagicDraw platform docs often still apply directly to Cameo plugin development.
- `.mdzip` is a local project package, not a complete substitute for repository semantics.
